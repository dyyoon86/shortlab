# 영상 대본: "안티그래비티 무료버전으로 나만의 웹페이지 만들어서 사용하기"

- 형식: 실습형 튜토리얼 (화면 녹화 + 내레이션)
- 타깃: 코딩 처음 접하는 비개발자
- 톤: 친절하고 쉬운 설명, 전문용어 나올 때마다 바로 풀어서 설명
- 구성: 안티그래비티(무료) → 프로젝트 실습 → GitHub 업로드 → Cloudflare Pages 연결 → 내 웹사이트 완성

---

## 0. 오프닝 훅 (0:00 ~ 0:20)

**[화면]** 완성된 shorts-lab 웹사이트 화면을 먼저 보여준다 (다크 랜딩 페이지 + AI 성우 페이지 시연)

**[내레이션]**
"이거, 코드 한 줄도 몰라도 오늘 다 따라 만들 수 있어요. AI 성우 기능에 영상 자막 자동 추출까지 들어있는 제 웹사이트인데, 이거 만드는 데 돈 한 푼도 안 들었습니다. 오늘은 구글의 '안티그래비티'라는 무료 AI 개발 툴 하나로, 여러분만의 웹페이지를 직접 만들고, 깃허브에 올리고, 클라우드플레어에 연결해서 실제로 인터넷에 서빙까지 시켜보는 걸 처음부터 끝까지 같이 해볼게요."

---

## 1. 문제 제기 (0:20 ~ 0:45)

**[화면]** "나만의 웹사이트 만들기"를 검색했을 때 나오는 복잡한 개발 용어들, 강의 목록 화면 캡처

**[내레이션]**
"근데 막상 찾아보면 프레임워크가 어쩌고, 배포가 어쩌고... 용어부터 막히죠. 저도 처음엔 그랬어요. 근데 요즘은 안티그래비티 같은 AI 코딩 툴이 대신 코드를 다 짜주기 때문에, 우리는 '뭘 만들고 싶은지'만 정확히 말해주면 됩니다. 오늘은 아무도 잘 모르는 이 조합 — 안티그래비티 무료버전 + 깃허브 + 클라우드플레어 페이지 — 이 세 개만으로 실습해볼 거예요. 셋 다 무료입니다."

---

## 2. 안티그래비티란? (0:45 ~ 1:15)

**[화면]** 안티그래비티 실행 화면, 채팅창에 프롬프트 입력하는 모습

**[내레이션]**
"안티그래비티는 구글에서 내놓은 AI 코딩 에이전트예요. 쉽게 말하면, 채팅으로 '이런 웹사이트 만들어줘'라고 하면 AI가 알아서 파일 만들고, 코드 짜고, 화면까지 확인해주는 도구입니다. 무료 버전만으로도 오늘 실습하는 정도는 충분히 커버됩니다."

**[화면 지시]** 안티그래비티 다운로드/설치 화면 or 웹 접속 화면 캡처 (실제 진행 시점 화면으로 교체)

---

## 3. 오늘 만들 것 소개 (1:15 ~ 1:45)

**[화면]** shorts-lab 프로젝트 스펙 문서(PROJECT.md) 화면, 3개 기능 카드 미리보기

**[내레이션]**
"오늘 실습으로 만드는 건 '쇼츠 제작 도우미' 사이트예요. 기능은 크게 세 개입니다. 첫째, 텍스트 넣으면 AI 목소리로 읽어주는 AI 성우. 둘째, 영상을 올리면 그 안의 말소리를 그대로 한국어 자막으로 뽑아주는 자막 추출 도구. 셋째, 영상에서 무음 구간을 자동으로 잘라주는 무음제거 도구. 이 세 개를 하나씩 순서대로 붙여나갈 거예요."

---

## 4. STEP 1 — 안티그래비티에게 요청하기 (1:45 ~ 2:45)

**[화면]** 안티그래비티 채팅창에 프롬프트 타이핑하는 화면 (실제 타이핑 과정 녹화)

**[내레이션]**
"이제 진짜 실습 들어갑니다. 첫 번째, 안티그래비티한테 뭘 만들고 싶은지 그대로 말해줄 거예요. '다크 테마 랜딩페이지 하나 만들어줘. 카드형으로 기능 3개를 보여주고, 각각 클릭하면 개별 페이지로 이동하게 해줘' 이런 식으로요. 중요한 건 화려한 말이 아니라, 뭘 만들지 구체적으로 적어주는 겁니다."

**[화면 지시]** 안티그래비티가 파일을 생성하고 코드를 작성하는 과정 타임랩스로 편집

**[내레이션]**
"이렇게 요청하면 안티그래비티가 알아서 페이지 파일들을 만들고, 코드를 짜고, 화면까지 미리 보여줍니다. 우리는 결과만 확인하면서 '여기 색깔 좀 바꿔줘' '이 버튼 크기 키워줘' 이런 식으로 대화하듯 수정하면 돼요."

---

## 5. STEP 2 — 결과 확인 & 다듬기 (2:45 ~ 3:15)

**[화면]** 로컬 미리보기 화면, 실제 랜딩페이지가 뜨는 모습

**[내레이션]**
"자, 벌써 이렇게 웹페이지가 하나 완성됐죠. 아직 제 컴퓨터 안에서만 보이는 상태고요. 이걸 이제 인터넷에 실제로 띄우는 작업, 즉 '배포'를 해야 다른 사람들도 접속할 수 있어요."

---

## 6. STEP 3 — 깃허브에 올리기 (3:15 ~ 4:15)

**[화면]** GitHub 사이트에서 새 저장소(New repository) 만드는 화면 → 안티그래비티(또는 터미널)에서 git init, add, commit, push 명령 실행 화면

**[내레이션]**
"먼저 깃허브에 새 저장소를 하나 만들어줍니다. 이름은 아무거나 상관없어요, 저는 'shorts-lab'이라고 지었어요. 그 다음 안티그래비티한테 '이 프로젝트를 방금 만든 깃허브 저장소에 올려줘'라고 하면, 코드를 저장하고 올리는 명령어까지 알아서 실행해줍니다. 우리는 깃허브 로그인이랑 저장소 이름만 준비하면 돼요."

**[화면 지시]** 새로고침한 GitHub 저장소 페이지에 파일들이 올라와 있는 모습 강조

---

## 7. STEP 4 — 클라우드플레어 페이지 연결 (4:15 ~ 5:15)

**[화면]** Cloudflare 대시보드 → Pages → "Connect to Git" 클릭 → 방금 만든 깃허브 저장소 선택 → 배포 설정 화면

**[내레이션]**
"이제 마지막 단계예요. 클라우드플레어라는 무료 호스팅 서비스에 들어가서, 'Pages'라는 메뉴에서 방금 만든 깃허브 저장소를 연결만 해주면 됩니다. 빌드 설정은 기본값 그대로 두고 '배포' 버튼만 누르면, 몇 분 안에 실제 인터넷 주소가 하나 생겨요."

**[화면 지시]** 배포 완료 후 나오는 실제 URL 클릭 → 브라우저에서 사이트가 뜨는 순간을 강조 (임팩트 컷)

**[내레이션]**
"짜잔, 이제 이 주소로 전 세계 누구나 제가 만든 페이지에 접속할 수 있어요. 그리고 제일 좋은 건, 앞으로 안티그래비티한테 '기능 하나 더 추가해줘' 하고 깃허브에 다시 올리기만 하면, 클라우드플레어가 자동으로 새 버전을 다시 배포해줍니다. 매번 이 과정을 반복할 필요가 없어요."

---

## 8. 마무리 & 다음 예고 (5:15 ~ 5:45)

**[화면]** 완성된 사이트 전체 화면 스크롤 + 세 가지 기능 카드 다시 비춤

**[내레이션]**
"오늘은 안티그래비티 무료버전, 깃허브, 클라우드플레어 페이지 이 세 가지 무료 도구만으로 나만의 웹사이트를 처음부터 배포까지 해봤습니다. 다음 편에서는 오늘 만든 사이트에 첫 번째 기능인 'AI 성우'를 실제로 붙여볼 거예요. 코드 몰라도 따라올 수 있으니까, 다음 편도 기대해주세요."

**[화면 지시]** 구독/알림 유도 자막 삽입
