# 영상 대본 — "코딩 몰라도 웹사이트 만들어서 인터넷에 띄우기" (안티그래비티 + 깃허브 + 버셀)

- **형식**: 실습형 튜토리얼 (화면 녹화 + 내레이션)
- **길이**: 약 6~7분
- **타깃**: 코딩 1도 모르는 완전 초보
- **원칙**: 어려운 용어 금지. 꼭 필요하면 바로 뒤에 쉬운 말로 한 번 더 풀어준다.
- **동반 교재**: 전자책 "코딩 몰라도 웹사이트 만들기 — 가입부터 배포까지" (사이트 `/ebooks`에서 열람)
- **배포처**: Vercel (기존 Cloudflare 버전에서 변경됨)
- **표기**: `[화면]` = 화면에 보이는 것 / `[내레이션]` = 말하는 대사 / `[자막]` = 화면 강조 자막

**[자막]** 코딩 0 · 돈 0 · 30분

## 1. 오늘 뭘 할 건지 (0:20 ~ 0:50)

**[화면]** 큰 글씨로 3단계 그림. `① 만든다 → ② 저장한다 → ③ 띄운다`

**[내레이션]**
"오늘 할 일은 딱 세 가지예요. 첫째, 'AI한테 말로 부탁해서' 웹사이트를 만들어요. 둘째, 만든 걸 인터넷 창고 같은 곳에 저장해요. 셋째, 그걸 진짜 인터넷 주소로 띄워서 아무나 들어올 수 있게 만들어요. 이 세 개만 하면 끝입니다."

**[화면]** 도구 3개 로고를 순서대로. `🪄 안티그래비티 · 📦 깃허브 · 🚀 버셀`

**[내레이션]**
"쓰는 도구도 세 개인데, 식당 차리는 걸로 비유하면 훨씬 쉬워요. 안티그래비티는 내가 '이런 메뉴 만들어줘' 말만 하면 척척 요리해주는 주방장이에요. 깃허브는 그 레시피랑 재료를 넣어두는 창고, 냉장고고요. 버셀은 드디어 가게 간판 달고 문을 여는 거예요. 이제 손님이, 그러니까 전 세계 누구나 들어올 수 있는 거죠. 메뉴 만들고, 창고에 넣고, 가게 열고. 딱 이 세 개예요."

> **비유는 대본 전체에 이어짐**: '오픈 전 = 주방에서만 맛보는 상태'(4장) → '가게 문 열기'(6장) → '메뉴 바꾸면 손님상에 자동 반영'(7장).
>
> **대체안 (원하는 걸로 교체)**
> - **게임(바람의나라·리니지) 버전**: "안티그래비티는 나 대신 아이템까지 만들어주는 친구, 깃허브는 그걸 넣어두는 '개인 창고', 버셀은 그 캐릭터가 사는 '서버'를 열어서 다른 사람도 접속해 볼 수 있게 해주는 거예요."
> - **심플(비유 없이)**: "만들어주는 AI '안티그래비티', 저장하는 창고 '깃허브', 인터넷에 띄워주는 '버셀'. 이름은 몰라도 되고, 각각 뭘 해주는지만 기억하면 됩니다."

**[내레이션]**
"먼저 안티그래비티. 주소창에 antigravity.google 치고, 다운로드 눌러서 설치해요. 다 깔고 처음 켜면 로그인 화면이 나오는데, '구글 계정으로 로그인' 눌러서 평소 쓰는 지메일로 들어가면 끝. 무료로 쓸 수 있는 양이 넉넉해서 오늘 실습은 공짜로 충분해요."

**[내레이션]**
"다음은 깃허브. 이건 내가 만든 파일을 보관하는 무료 창고예요. github.com 가서 오른쪽 위 'Sign up'. 이메일, 비밀번호, 그리고 사용자 이름을 정해요. 이 이름은 나중에 주소에 들어가니까 영어로 짧게. 이메일로 온 인증 코드까지 넣으면 가입 끝."

**[자막]** 사용자 이름 · 비밀번호는 메모해두기!

"마지막 가입, 버셀. 이건 창고에 있는 걸 진짜 인터넷 주소로 띄워주는 곳이에요. vercel.com 가서 회원가입 누르고, '깃허브로 계속하기'를 골라요. 방금 만든 깃허브 계정을 그대로 쓰는 거예요. 권한 허용 버튼 한 번 눌러주고, 개인용 무료인 'Hobby' 선택하면 세 개 가입 전부 끝!"

## 3. STEP 1 — AI한테 말로 부탁하기 (2:10 ~ 3:20)

**[화면]** 안티그래비티 채팅창에 프롬프트를 타이핑.

**[내레이션]**
"자 이제 진짜 만들어볼게요. 안티그래비티 채팅창에다가, 만들고 싶은 걸 그냥 우리말로 적으면 돼요. 이 부탁하는 글을 '프롬프트'라고 불러요. 어려운 단어 안 써도 됩니다. 눈에 보이듯이 자세히 설명하는 게 전부예요."

**[화면 / 자막]** 실제로 붙여넣는 프롬프트를 화면에 크게:

> 쇼츠 만드는 걸 도와주는 웹사이트를 만들어줘.
> 어두운 배경에 카드 3개가 있는 첫 화면으로.
> 카드를 누르면 각각 다른 페이지로 넘어가게 해줘.
> Vite랑 React로 만들고, 나중에 깃허브랑 버셀에 배포할 거야.

**[화면 지시]** 엔터 → AI가 파일을 만들고 미리보기를 보여주는 과정을 빠르게 편집(타임랩스).

**[내레이션]**
"이렇게 부탁하고 엔터를 누르면, AI가 알아서 화면이랑 기능을 다 만들어주고 결과를 바로 보여줘요. 중간에 '이렇게 할까요?' 하고 물어보면 '응, 그렇게 해줘' 하면 됩니다. 마음에 안 드는 데 있으면 '여기 색깔 바꿔줘', '글씨 크게 해줘' 이렇게 카톡하듯이 계속 고쳐달라고 하면 돼요."

## 5. STEP 3 — 깃허브 창고에 저장하기 (push) (3:45 ~ 4:40)

**[화면]** 안티그래비티 채팅창에 저장 요청 프롬프트 입력.

**[내레이션]**
"만든 걸 창고에 올리는 걸 '푸시'라고 해요. 파일을 창고로 밀어 넣는다는 뜻이에요. 이것도 어렵지 않아요. 안티그래비티한테 그냥 시키면 됩니다."

**[화면 / 자막]** 프롬프트 크게:

> 지금까지 만든 걸 깃허브에 올려줘.
> 저장소 이름은 shorts-lab, 공개로 만들고,
> node_modules 폴더는 빼고 push 해줘.

**[화면 지시]** AI가 깃허브 권한 허용 받고 push 하는 과정 → 깃허브 사이트 새로고침하니 파일들이 창고에 들어와 있는 화면.

**[내레이션]**
"이렇게 부탁하면, 중간에 깃허브 로그인이나 권한을 물어볼 수 있어요. 화면 안내대로 허용만 해주면 됩니다. 다 되면 깃허브 사이트에 들어가서 내 창고를 열어보세요. 방금 만든 파일들이 쫙 들어와 있으면 성공이에요."

## 6. STEP 4 — 버셀로 진짜 인터넷에 띄우기 (4:40 ~ 5:50)

**[화면]** `vercel.com/new` 접속 → 깃허브 창고 목록에서 `shorts-lab` 옆 **Import** 클릭.

**[내레이션]**
"마지막 단계예요. 버셀에 들어가서 vercel.com 슬래시 new, 여기로 가면 내 깃허브 창고 목록이 보여요. 아까 만든 shorts-lab 옆에 'Import', 가져오기 버튼을 눌러요. 만약 창고가 안 보이면, 권한을 열어주는 버튼이 있으니 그것만 눌러주면 나타나요."

**[화면]** 빌드 설정 화면. `Framework: Vite`, `Build Command: npm run build`, `Output Directory: dist` 를 강조.

**[내레이션]**
"그러면 설정 화면이 나오는데, 거의 다 자동으로 맞춰져 있어요. 프레임워크가 'Vite', 이렇게만 돼 있으면 됩니다. 나머지는 건드릴 필요 없어요. 그냥 큰 'Deploy', 배포 버튼을 눌러요."

**[화면 지시]** 로그가 지나가고 몇 분 뒤, 축하 화면과 함께 `*.vercel.app` 주소가 뜨는 순간을 크게 강조. 그 주소를 클릭하니 사이트가 뜨는 장면.

**[내레이션]**
"버튼을 누르면 화면에 글자가 주르륵 지나가는데, 겁먹지 마세요. 이건 가게가 우리가 준 레시피랑 재료로 진짜 요리를 완성하는 중인 거예요. 1분에서 3분 정도 기다리면, 짜잔! 축하 폭죽과 함께 진짜 인터넷 주소가 하나 나와요. 요리가 다 돼서 드디어 가게 문을 여는 순간이에요. 이 주소를 누르면 — 보세요, 제가 만든 사이트가 진짜 인터넷에 떴습니다. 이제 이 주소만 있으면 전 세계 누구나, 친구 폰에서도 손님처럼 이 페이지에 들어올 수 있어요."

**[자막]** 내 사이트가 인터넷에 공개됨 🎉

## 7. 보너스 — 이제부터가 진짜 편해요 (5:50 ~ 6:20)

**[화면]** `안티그래비티로 고침 → 깃허브에 push → 버셀이 자동으로 새로 띄움` 그림.

**[내레이션]**
"그리고 진짜 좋은 점. 한 번 이렇게 가게를 열어두면, 앞으로는 버셀을 다시 건드릴 필요가 없어요. 안티그래비티한테 '기능 하나 더 추가해줘' 부탁하고, '깃허브에 올려줘' 하면 그걸로 끝이에요. 그다음은 버셀이 알아서 해요. 새 레시피랑 재료가 창고에 들어온 걸 보고, 아까처럼 가게가 알아서 다시 요리를 하고, 완성되면 손님상까지 새 메뉴로 자동으로 바꿔줘요. 우리는 바뀐 레시피랑 재료만 창고에 새로 넣어주면, 요리부터 손님상에 내는 것까지 전부 가게가 알아서 해주는 거죠. 매번 우리가 요리하고 문 열고 할 필요가 없어요."
