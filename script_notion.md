# 🎬 코딩 몰라도 웹사이트 만들어서 인터넷에 띄우기

> 안티그래비티 + 깃허브 + 버셀 | 실습형 튜토리얼 (화면 녹화 + 내레이션)

| 항목 | 내용 |
| --- | --- |
| 길이 | 약 6~7분 |
| 타깃 | 코딩 1도 모르는 완전 초보 (40~50대) |
| 톤 | 쉬운말, "식당 차리기" 비유로 관통 |
| 배포처 | Vercel |
| 동반 교재 | 전자책 "코딩 몰라도 웹사이트 만들기" (사이트 `/ebooks`) |

표기: 🎬 = 화면 / 🎙️ = 내레이션 / 💬 = 자막

---

## 0. 오프닝 훅 (0:00~0:20)

🎬 완성된 사이트 시연 — 카드 눌러 이동, 글자 넣고 AI 목소리 재생.

> 🎙️ "이거 지금 화면에 나오는 웹사이트, 제가 직접 만든 거예요. 근데 저 코딩 하나도 몰라요. 글자를 넣으면 AI 목소리로 읽어주는 기능까지 있고요. 오늘은 이걸 처음부터 끝까지, 진짜 쉽게 만들어서 인터넷에 띄우는 것까지 다 보여드릴게요. 돈도 하나도 안 들어요."

💬 코딩 0 · 돈 0 · 30분

---

## 1. 오늘 뭘 할 건지 (0:20~0:50)

🎬 3단계 그림: `① 만든다 → ② 저장한다 → ③ 띄운다` / 도구 3개 로고.

> 🎙️ "오늘 할 일은 딱 세 가지예요. 첫째, AI한테 말로 부탁해서 웹사이트를 만들어요. 둘째, 만든 걸 인터넷 창고에 저장해요. 셋째, 진짜 인터넷 주소로 띄워서 아무나 들어올 수 있게 만들어요."

> 🎙️ "쓰는 도구도 세 개인데, **식당 차리는 걸로** 비유하면 훨씬 쉬워요. **안티그래비티**는 '이런 메뉴 만들어줘' 말만 하면 척척 요리해주는 **주방장**. **깃허브**는 그 레시피랑 재료를 넣어두는 **창고·냉장고**. **버셀**은 드디어 가게 간판 달고 문을 여는, 손님을 받는 **가게**예요. 메뉴 만들고, 창고에 넣고, 가게 열고. 딱 이 세 개예요."

---

## 2. 준비물 & 가입 (0:50~2:10)

> 🎙️ "준비물은 딱 하나, 평소 쓰는 구글 계정. 이제 도구 세 개에 가입만 하면 돼요."

### 2-1. 안티그래비티

🎬 `antigravity.google` → **Download** → 설치 → **구글 계정으로 로그인**

> 🎙️ "먼저 안티그래비티. antigravity.google 에서 다운로드 눌러 설치하고, 처음 켜면 '구글 계정으로 로그인'. 무료로 쓸 수 있는 양이 넉넉해서 오늘 실습은 공짜로 충분해요."

### 2-2. 깃허브

🎬 `github.com` → 오른쪽 위 **Sign up** → 이메일·비번·사용자이름 → 인증코드

> 🎙️ "다음은 깃허브. 내가 만든 파일을 보관하는 무료 창고예요. 오른쪽 위 'Sign up' 눌러서 이메일, 비밀번호, 사용자 이름 정하고, 이메일 인증코드까지 넣으면 끝. 사용자 이름은 나중에 주소에 들어가니 영어로 짧게."

💬 사용자 이름·비밀번호는 메모!

### 2-3. 버셀

🎬 `vercel.com` → **Sign Up** → **Continue with GitHub** → **Authorize** → Hobby(무료)

> 🎙️ "마지막 버셀. 창고에 있는 걸 진짜 인터넷 주소로 띄워주는 곳이에요. 회원가입에서 '깃허브로 계속하기'를 고르면, 방금 만든 깃허브 계정을 그대로 써요. 권한 허용 한 번 누르고 개인용 무료 'Hobby' 선택하면 세 개 가입 끝!"

---

## 3. STEP 1 — AI한테 말로 부탁하기 (2:10~3:20)

🎬 안티그래비티 채팅창에 프롬프트 타이핑.

> 🎙️ "만들고 싶은 걸 그냥 우리말로 적으면 돼요. 이 부탁하는 글을 '프롬프트'라고 해요. 눈에 보이듯이 자세히 설명하는 게 전부예요."

💬 붙여넣는 프롬프트:

```
쇼츠 만드는 걸 도와주는 웹사이트를 만들어줘.
어두운 배경에 카드 3개가 있는 첫 화면으로.
카드를 누르면 각각 다른 페이지로 넘어가게 해줘.
Vite랑 React로 만들고, 나중에 깃허브랑 버셀에 배포할 거야.
```

🎬 (타임랩스) AI가 파일 만들고 미리보기 보여줌.

> 🎙️ "엔터를 누르면 AI가 알아서 화면이랑 기능을 다 만들어주고 결과를 바로 보여줘요. '이렇게 할까요?' 물어보면 '응, 그렇게 해줘'. 마음에 안 들면 '여기 색깔 바꿔줘' 이렇게 카톡하듯이 계속 고쳐달라고 하면 돼요."

---

## 4. STEP 2 — 아직은 오픈 전 (3:20~3:45)

🎬 내 컴퓨터 안 미리보기.

> 🎙️ "벌써 화면이 만들어졌죠. 근데 이건 아직 제 컴퓨터 안에서만 보여요. 식당으로 치면 **주방에서 저 혼자 맛보는 오픈 전 상태**예요. 손님한테 내놓으려면 두 단계가 남았어요. 창고에 넣고, 가게 문 열기."

---

## 5. STEP 3 — 깃허브 창고에 저장 (push) (3:45~4:40)

🎬 안티그래비티 채팅창에 저장 요청.

> 🎙️ "만든 걸 창고에 올리는 걸 '푸시'라고 해요. 이것도 안티그래비티한테 시키면 됩니다."

💬 붙여넣는 프롬프트:

```
지금까지 만든 걸 깃허브에 올려줘.
저장소 이름은 shorts-lab, 공개로 만들고,
node_modules 폴더는 빼고 push 해줘.
```

🎬 권한 허용 → push → 깃허브 새로고침하니 파일 들어와 있음.

> 🎙️ "중간에 깃허브 로그인/권한을 물어보면 화면 안내대로 허용만 해주면 됩니다. 다 되면 깃허브 창고에 파일들이 쫙 들어와 있으면 성공이에요."

---

## 6. STEP 4 — 버셀로 가게 문 열기 (배포) (4:40~5:50)

🎬 `vercel.com/new` → 창고 목록에서 `shorts-lab` 옆 **Import**.

> 🎙️ "vercel.com 슬래시 new 로 가면 내 깃허브 창고 목록이 보여요. shorts-lab 옆 'Import'를 눌러요. 안 보이면 권한 열어주는 버튼만 누르면 나타나요."

🎬 빌드 설정: `Framework: Vite` 강조 → **Deploy**.

> 🎙️ "설정은 거의 자동이에요. 프레임워크가 'Vite'로 돼 있으면 그냥 큰 'Deploy' 버튼을 눌러요."

🎬 로그 지나감 → 몇 분 뒤 `*.vercel.app` 주소 발급 → 클릭하니 사이트 뜸.

> 🎙️ "버튼을 누르면 화면에 글자가 주르륵 지나가는데, 이건 **가게가 우리 레시피랑 재료로 진짜 요리를 완성하는 중**이에요. 1~3분 뒤 짜잔! 진짜 인터넷 주소가 나와요. **드디어 가게 문을 여는 순간**이에요. 이 주소만 있으면 전 세계 누구나, 친구 폰에서도 손님처럼 들어올 수 있어요."

💬 내 사이트가 인터넷에 공개됨 🎉

---

## 7. 보너스 — 이제부터가 진짜 편해요 (5:50~6:20)

🎬 `레시피·재료 고쳐서 push → 가게가 자동으로 다시 요리 → 손님상에 새 메뉴 자동 반영` 그림.

> 🎙️ "한 번 가게를 열어두면 앞으로 버셀은 다시 안 건드려도 돼요. 안티그래비티한테 '기능 추가해줘' 하고 '깃허브에 올려줘' 하면 끝. 그다음은 버셀이 알아서, **새 재료가 창고에 들어온 걸 보고 다시 요리해서 손님상까지 새 메뉴로 자동으로 바꿔줘요.** 우리는 재료만 새로 넣어주면 되는 거죠."

---

## 8. 마무리 (6:20~6:50)

🎬 완성 사이트 + `/ebooks` 서재 페이지.

> 🎙️ "오늘은 안티그래비티, 깃허브, 버셀. 이 세 가지 공짜 도구만으로 웹사이트를 만들고, 저장하고, 인터넷에 띄우는 것까지 다 해봤어요. 오늘 내용은 전자책으로도 정리해뒀어요. 사이트의 '전자책 서재'에서 보실 수 있습니다. 다음 영상에서는 여기에 '목소리 읽어주는 기능'을 넣어볼게요. 다음 편도 기대해주세요!"

💬 구독/알림 + 전자책 링크

## 부록 A. 숏폼 자막 (하단 텍스트)

1. 코딩 몰라도 웹사이트 만들 수 있어요
2. AI한테 말로 부탁하면 끝
3. 만든 거 무료 창고(깃허브)에 저장
4. 버셀로 버튼 몇 번 → 인터넷에 공개
5. 돈 한 푼도 안 들어요
6. 고칠 땐 저장만 하면 자동 반영
7. 다음편: 목소리 읽어주는 기능 넣기
