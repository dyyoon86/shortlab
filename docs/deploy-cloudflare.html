자동 배포하기

이 저장소는 이미 Cloudflare Pages용으로 세팅이 끝나 있습니다. 코드 수정 없이 아래 순서대로 연동만 하면, master 브랜치에 push할 때마다 사이트가 자동으로 배포됩니다.

✳ 왜 Vercel이 아니라 Cloudflare인가

TTS(AI 성우) 기능 때문에 반드시 Cloudflare여야 합니다.

functions/api/tts.ts 는 MS Edge의 비공식 TTS 서버에 아웃바운드 WebSocket으로 붙습니다. 이 기능은 Cloudflare Workers 런타임에서만 동작하기 때문에, Vercel 등 다른 플랫폼에서는 TTS가 작동하지 않습니다. 나머지 두 기능(자막 추출·무음 제거)은 방문자 브라우저에서 실행되는 완전 정적 기능이라 서버가 필요 없습니다.

💰

비용은 $0. Pages Functions 무료 플랜은 하루 10만 요청까지 지원하며, 이 퍼널용 사이트 규모에는 충분합니다.

1 연동 시 입력할 빌드 설정

Cloudflare가 이 값들로 저장소를 빌드합니다. 그대로 입력하세요.

Production branchmaster  ⚠️ main 아님
Framework presetVite
Build commandnpm run build
Build output directorydist
Root directory(비움)
📁

Functions는 자동 인식됩니다. functions/ 폴더가 있으면 Cloudflare Pages가 알아서 /api/tts 서버리스 함수로 배포하므로, 별도 설정이 필요 없습니다.

2 대시보드에서 연동하기

약 3분. Cloudflare 계정만 있으면 됩니다.

  1. Cloudflare Dashboard 접속 → 왼쪽 메뉴 Workers & Pages.
  2. Create → Pages 탭 → Connect to Git 클릭.
  3. GitHub 계정 인증 후 저장소 목록에서 dyyoon86/shortlab 선택.
  4. 위 1번 표의 빌드 설정 값을 그대로 입력. (Production branch가 master인지 꼭 확인)
  5. Save and Deploy 클릭 → 첫 배포 시작. 1~2분 뒤 https://shortlab.pages.dev 형태의 URL이 생성됩니다.
🔁

이후는 완전 자동. master 브랜치에 push할 때마다 Cloudflare가 자동으로 다시 빌드·배포합니다. 대시보드를 다시 열 필요 없습니다.

3 배포 후 확인 체크리스트

아래 4가지가 되면 연동 성공입니다.

  • 배포 URL 접속 → 랜딩 페이지가 정상적으로 표시된다
  • /tts 페이지 → 음성 생성 버튼 → mp3가 재생된다  (← 이게 되면 Functions 연동 성공)
  • /subtitle → 영상 업로드 후 자막 추출 (첫 로딩 시 모델 다운로드로 느림 = 정상)
  • /silence → 무음 구간 제거 결과물 다운로드
🎙️

만약 /tts만 실패한다면 설정 문제가 아니라 edge-tts(비공식 MS API)가 일시적으로 막힌 것입니다. 나머지 페이지가 정상이면 배포 자체는 성공한 것입니다.

4 로컬 테스트 & 문제 해결

배포 전에 로컬에서 미리 확인하고 싶다면.

일반 개발 서버 (TTS 함수 제외)

npm install
npm run dev

Cloudflare 환경 그대로 (TTS 함수 포함) 로컬 재현

npm run dev:cf   # vite build + wrangler pages dev dist
🟢

빌드가 실패하면 Node 버전 문제일 수 있습니다. Cloudflare Pages 프로젝트 설정 → Environment variables 에 NODE_VERSION = 20 을 추가하세요.

5 내 도메인 붙이기 (선택)

배포된 Pages 프로젝트 → Custom domains 탭 → Set up a custom domain → 원하는 도메인 입력. 도메인이 Cloudflare에서 관리 중이면 DNS가 자동 연결되고, 외부 도메인이면 안내되는 CNAME 레코드만 추가하면 됩니다.
