# shorts-lab

쇼츠 제작 도우미 사이트. [junhwi-shorts.vercel.app](https://junhwi-shorts.vercel.app/) 구조를 참고해서 새로 만드는 개인 프로젝트. 콘텐츠(도구)는 한 번에 다 만들지 않고 하나씩 추가해나간다. 이 문서는 Antigravity로 개발 착수할 때 참고할 스펙 문서.

## 배포 파이프라인

- GitHub 저장소 → master 브랜치 push 시 **Vercel** 자동배포
- 프레임워크: Vite + React (SSR/서버 상태 없음, 정적 산출물 + 서버리스 함수 1개)
- **배포처가 Vercel인 이유**: AI 성우(TTS)가 edge-tts에 **아웃바운드 웹소켓**으로 붙는데, Cloudflare Workers 런타임(workerd)에서는 이 웹소켓이 핸드셰이크(101)까지만 되고 오디오 데이터를 못 받고 끊긴다(실측 검증 완료). 진짜 Node.js 런타임인 Vercel 서버리스 함수에서는 표준 `ws` 클라이언트로 정상 동작한다. → 초기엔 Cloudflare Pages로 시도했다가 이 문제로 **Vercel로 이전**함.
- 스타일: 레퍼런스 사이트와 유사한 다크 테마 랜딩 + 카드형 기능 목록 → 각 기능 개별 페이지

## 레퍼런스 사이트 구조 (참고용, 실제 확인함)

랜딩 페이지: 로고 + 타이틀("쇼츠를 만드는 모든 도구") + 5개 기능 카드(맞춤 로드맵 / AI 성우 / 일본 체험판 / 전자책 서재 / 캥거루 점프) + "시작하기" CTA.

`/tts` 페이지 구조 (AI 성우):
1. 언어 선택 (한/일/영/중)
2. 성우 선택 (성별 + 톤 설명이 붙은 카드형 프리셋 2~3개)
3. 대본 입력 (텍스트박스, 글자수 제한 표시)
4. 속도·톤 슬라이더
5. "음성 생성하기" 버튼 → 서버 호출 → mp3 응답

이 사이트는 버튼 클릭 시 `/api/tts`를 호출해서 `audio/mpeg`를 직접 받는 구조로 확인됨 (= MS Edge 비공식 TTS를 서버에서 대신 호출해주는 프록시 방식).

## 디자인 시스템 (레퍼런스에서 실제 추출)

브라우저 개발자 도구로 레퍼런스 사이트의 실제 CSS(Tailwind 클래스)를 뽑아서 확인함. 아래 값 그대로 재사용.

- **배경**: 전체 화면 다크 네이비 대각선 그라데이션, 흐르듯 움직이는 애니메이션
  `bg-gradient-to-br from-[#04081a] via-[#0d1530] to-[#04081a]` + `animate-gradient-shift`
- **배경 장식**: 화면 모서리에 큰 원형 블러(glow orb) 2개
  우상단 `bg-cyan-400/25 blur-3xl rounded-full`, 좌하단 `bg-amber-300/20 blur-3xl rounded-full`
- **폰트**: Pretendard Variable (없으면 Apple SD Gothic Neo → Noto Sans KR → 시스템 폰트 순 폴백)
- **타이틀 강조 텍스트**: 그라데이션 + 반짝이는 애니메이션
  `bg-gradient-to-r from-cyan-300 via-amber-200 to-pink-300 bg-clip-text text-transparent` + `animate-shimmer-text`
- **메인 CTA 버튼**: 노랑~호박색 그라데이션, 진한 네이비 글자, 완전 둥근 모서리
  `bg-gradient-to-r from-amber-300 to-yellow-200 hover:from-yellow-200 hover:to-amber-300 text-[#0d1530] font-black rounded-2xl px-8 py-4`
- **기능 카드**: 반투명 유리(glassmorphism) 느낌 — 흰색 3% 배경 + 흰색 10% 테두리, 둥근 모서리
  `bg-white/[0.03] border border-white/10 rounded-[17px]`
- **작은 뱃지/필**: 완전 둥근 알약 모양, 옅은 시안 배경
  `rounded-full bg-cyan-300/10`

**그대로 안 가져올 것**: 캥거루 마스코트 로고, "핸드인캥거루"/강사 브랜딩 요소는 레퍼런스 고유 브랜드이므로 우리 로고·문구로 교체. 색감·타이포·카드/버튼 스타일·배경 그라데이션과 글로우 연출만 그대로 재사용.

## 이번에 만드는 기능 3개

### 1. AI 성우 (TTS)

- **최종 결정: edge-tts + Vercel 서버리스 함수 1개(`api/tts.ts`, Node.js + `ws`) 사용.** (검토했던 ~~Kokoro-82M~~ → ~~MMS-TTS~~ → ~~Web Speech API~~ → ~~Cloudflare Pages Function~~ 은 전부 폐기)
- **변경 이유**: 브라우저에서 직접 신경망 모델을 돌리는 방식(MMS-TTS)은 느리고, 브라우저 내장 음성(Web Speech API)은 빠르지만 기기당 한국어 음성이 보통 1개뿐이라 성우 다양성이 안 나옴. 반면 edge-tts는 레퍼런스 사이트 실측 기준 응답 450ms로 빠르고, 언어별로 진짜 다양한 성우(성별/톤 다른 여러 명)를 제공함. 유일한 단점이던 "서버 필요"는 Vercel 서버리스 함수(무료 Hobby 플랜)로 해결 — 개인 퍼널용 규모엔 무료 쿼터로 충분.
- **왜 Cloudflare가 아니라 Vercel?**: edge-tts는 웹소켓 프로토콜인데 Cloudflare Workers(workerd)에서는 핸드셰이크 후 데이터를 못 받음(검증). Node.js 런타임(Vercel)의 표준 `ws` 클라이언트에서는 정상. 자세한 근거는 상단 "배포 파이프라인" 참고.
- **엔진**: edge-tts (MS Edge 비공식 TTS)를 **Vercel 서버리스 함수 1개**가 서버에서 호출 → mp3 스트림으로 응답. 브라우저는 이 함수(`/api/tts`)만 호출하면 됨.
- **성우(보이스) 선택**: edge-tts가 언어별로 제공하는 실제 다중 음성 목록(성별/톤 다른 여러 프리셋) 그대로 사용 — 레퍼런스처럼 "선희/인준/현수" 식 카드형 성우 선택 UI 그대로 구현 가능. 가짜 피치 가공 필요 없음.
- 입력: 언어(ko/ja/en/zh), 성우 프리셋, 대본 텍스트, 속도/톤 슬라이더
- 출력: 서버리스 함수가 mp3 스트림 반환 → 바로 재생 + 다운로드
- 리스크: edge-tts는 비공식 API라 MS가 언제든 막을 수 있음(그레이존). 상업적 대량 사용은 지양, 개인/소규모 퍼널 용도로는 지금까지 문제없이 잘 도는 방식. (막히면 Azure Speech 공식 REST로 전환 — 같은 성우 사용 가능)
- **아키텍처 예외**: 이 저장소에서 서버(Vercel 서버리스 함수)가 필요한 건 이 기능 하나뿐. 나머지 두 기능은 여전히 완전 정적/클라이언트 사이드.

### 2. 영상 자막 추출 (한국어)

- **번역 기능은 뺌.** 일본어→한국어 번역 단계, 번역 모델(NLLB/M2M100) 전부 제거. Whisper 하나로만 처리.
- **완전 클라이언트 사이드 (서버 없음, 정적 페이지로 가능)**
- 파이프라인:
  1. 영상 파일 업로드 (브라우저 내 처리, 서버 업로드 없음)
  2. (선택) 무음 구간 제거 전처리 → 아래 3번 기능 재사용
  3. **Whisper** (transformers.js / onnxruntime-web, whisper-small 권장, task=transcribe, language=ko) → 영상 속 한국어 음성을 바로 한국어 자막 텍스트 + 타임스탬프로 변환
  4. 결과: SRT/VTT 자막 파일 다운로드 (+ 가능하면 영상 미리보기에 자막 오버레이)
- 번역 단계가 빠지면서 구조가 훨씬 단순해짐: 모델 하나(Whisper)만 다운로드하면 되고, 처리 속도도 더 빠름.
- 모델은 최초 방문 시 다운로드되고 IndexedDB 캐시(transformers.js 기본 동작)로 재방문 시 재사용됨.
- 트레이드오프: 품질/속도는 상용 API(Whisper API 등) 대비 낮을 수 있고, 저사양 기기·모바일에서 느릴 수 있음. WebGPU 지원 브라우저면 훨씬 빠름.

### 3. 무음 제거

- **완전 클라이언트 사이드, AI 모델 불필요** (순수 신호처리)
- Web Audio API로 오디오 파형 디코딩 → RMS/진폭 기준 임계값 이하 구간 탐지 → 해당 구간 컷/스킵
- 기능 2의 전처리 단계로도 쓰이고, 독립 도구로도 노출 (영상/오디오 업로드 → 무음 구간 제거된 결과물 다운로드)
- 저사양 기기에서도 가볍게 동작 (AI 추론 없음)

## 아키텍처 요약

| 기능 | 서버 필요 여부 | 사용 기술 |
|---|---|---|
| AI 성우 (TTS) | **필요** (Vercel 서버리스 함수 1개 `api/tts.ts`, Node.js+ws) | edge-tts 프록시 |
| 영상 자막 추출 (한국어) | 불필요 (완전 정적) | transformers.js: Whisper(small) |
| 무음 제거 | 불필요 (완전 정적) | Web Audio API |

→ 저장소는 대부분 정적이고, **AI 성우 기능 하나만** Vercel 서버리스 함수로 처리(무료 Hobby 쿼터로 충분). 나머지 두 기능은 서버 없이 방문자 브라우저에서 실행. 전체적으로 비용은 여전히 $0.
